Qu'est-ce que Docker ?

Docker est une plateforme de conteneurisation qui permet de packager une application avec toutes ses dépendances dans un conteneur isolé et portable.

info
Conteneur vs Machine virtuelle
Un conteneur partage le noyau de l'OS hôte, ce qui le rend beaucoup plus léger et rapide qu'une VM traditionnelle.

Concepts clés

  • Image — Un template immuable contenant l'OS, le runtime et l'application
  • Conteneur — Une instance d'exécution d'une image
  • Dockerfile — Un fichier texte décrivant comment construire une image
  • Registry — Un dépôt d'images (Docker Hub, GitHub Container Registry)

Installation

Vérifiez votre installation :

docker --version
# Docker version 24.0.7, build afdd53b

docker run hello-world

Commandes essentielles

bash 
# Lister les images
docker images

# Télécharger une image
docker pull nginx:alpine

# Lancer un conteneur
docker run -d -p 8080:80 --name mon-nginx nginx:alpine

# Lister les conteneurs actifs
docker ps

# Voir les logs
docker logs mon-nginx

# Arrêter et supprimer
docker stop mon-nginx
docker rm mon-nginx

Premier Dockerfile

Créons une image pour une application Node.js :

dockerfile 
# Image de base
FROM node:20-alpine

# Répertoire de travail
WORKDIR /app

# Copier les fichiers de dépendances
COPY package*.json ./

# Installer les dépendances
RUN npm ci --only=production

# Copier le code source
COPY . .

# Port exposé
EXPOSE 3000

# Commande de lancement
CMD ["node", "server.js"]
lightbulb
Bonne pratique
Copiez d'abord package.json avant le reste du code. Docker met en cache chaque couche (layer), donc si vos dépendances n'ont pas changé, l'installation sera instantanée.
